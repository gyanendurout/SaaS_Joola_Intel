3. Server endpoints (Next.js API)

Four serverless POST endpoints. Each takes a JSON body, calls OpenAI, returns a structured response. No database writes — these are pure transformation/generation tools used inside the dashboard.

RoutePurposeExternal callReturns
POST /api/generate-content Generate marketing copy from a prompt + brand voice OpenAI Chat Completions (gpt-4o-mini) { content: string, tokens_used: number }
POST /api/keyword-research Suggest related search keywords for a seed term OpenAI (gpt-4o-mini) { keywords: string[], clusters: object[] }
POST /api/content-brief Produce a structured content brief (outline, headers, target audience) from a seed keyword + cluster OpenAI (gpt-4o-mini) via lib/shared/content-brief/ { title, audience, sections, faq, target_word_count }
POST /api/seo-analyzer On-page audit: fetch a URL, parse HTML, score it + recommend fixes cheerio (HTML parse) + OpenAI (recommendations) { score: 0–100, issues: [], recommendations: [] }

Important indirection: our Python code never hits Meta, Google, Instagram, etc. directly. Every "Source URL" above is the public page that an Apify actor scrapes on our behalf using Apify's residential proxies and managed browser pool. Our code talks to Apify; Apify talks to the platforms.

Total runtime: 30–90 min. Cost: ~$5–$20 per pass (Apify), ~$0.50–$2 (OpenAI enrichment). Resumes from pipeline_state.json if interrupted. Each step has 80 retries on transient errors (ReadTimeout, 5xx).

6. Apify actor catalog

All actors are public marketplace actors. None are custom-built — the pipeline drives them via the standard Apify API with input JSON and polls for completion.

Why Apify?

  • Residential proxies built-in (Instagram, TikTok, X aggressively block datacenter IPs).
  • 30-day dataset retention — easy to re-download a run if a write to Supabase fails.
  • Pay-per-run pricing — no monthly minimum.
  • Single auth token across all actors (APIFY_TOKEN).

How a step works

  1. Build input JSON (handles to scrape, time window, max items).
  2. POST /v2/acts/{actor}/runs → returns run ID.
  3. Poll GET /v2/actor-runs/{runId} every 5–15s until SUCCEEDED or FAILED.
  4. Fetch the run's dataset: GET /v2/datasets/{datasetId}/items.
  5. Transform + upsert into Supabase via PostgREST.

7. AI enrichment (OpenAI)

After raw rows land, scripts/pipeline/enrich_with_ai.py walks every unenriched row across ig_posts, ig_comments, youtube_videos, youtube_comments, reddit_mentions, reddit_comments, x_posts, tiktok_videos, and marketing_ads. One GPT-4o-mini call per row; results written back as 12 enrichment columns.

8. Derived tables — facts & lifecycle

mention_facts

Built by populate_mention_facts.py. Walks every enriched row and emits one fact-table row per (channel, source_url, entity) triple. This denormalization is what makes the dashboard reads fast — no joins needed.

Channels: ig_post, ig_comment, yt_video, yt_comment, reddit_op, reddit_comment, x_post, tiktok_video, ad.

topic_lifecycle

Built by populate_topic_lifecycle.py. For each detected topic, computes: first-seen channel + timestamp, peak-week mention count, decay rate, current week share. ~621 topics tracked across 8,007 occurrences (last full run).

Used by Market Intel page to show emerging topics and which channel typically breaks a trend first (often Reddit comments or TikTok, ahead of IG posts).
